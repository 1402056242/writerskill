# Novel Kit Constitution (创作宪法)

> **定位**：这是本项目的最高法律。所有生成的文本必须遵守以下原则。  
> **适用范围**：设定、大纲、正文、章节提交（chapter commits）、记忆文件（memory）。

---

## 1. 核心原则 (Core Principles)

* **原则一：爽点优先 (Dopamine First)**
  * 每一章必须有一个明确的情绪落点（期待 / 爽 / 惊讶 / 压抑后释放）。
  * 禁止无意义的水文。如果一段剧情不能推动主线或塑造人物，删掉它。

* **原则二：逻辑自洽 (Internal Logic)**
  * 金手指必须有代价或限制，不得无限白给、战力崩坏。
  * 反派必须有智商，决策符合其利益最大化，拒绝「为了给主角送经验」而降智。
  * 世界观设定一旦确立，不可随意吃书；如需修正，必须通过「设定回溯 + 重写章节」的方式，而不是假装没有发生。

* **原则三：人设一致性 (Character Consistency)**
  * 主角性格（如：苟 / 冷酷 / 腹黑）必须贯穿始终。
  * 严禁 OOC (Out Of Character)。主角不能为了推动剧情而做出与性格完全不符的蠢事。
  * 角色成长必须有足够事件与心理铺垫，不能凭空改变三观或立场。

---

## 2. 文风红线 (Style Guardrails)

* **严禁翻译腔**：禁止使用「哦，我的老伙计」、「难道不……吗？」等直译句式。
* **严禁过度描写**：环境描写服务于氛围，外貌描写服务于性格。单次环境描写不超过 **100 字**。
* **严禁词汇重复**：同一段落内禁止出现两次相同的形容词（如「恐怖如斯」），高频词要主动替换。

---

## 3. 状态与事实同步 (State & Fact Synchronization)

> 本项目使用 **Story System** 管理长篇事实：  
> `.novelkit/story/commits/chapter_{X}.commit.json` 是章节事实主链，  
> `.novelkit/memory/character_state.md` 与 `.novelkit/memory/foreshadowing.md` 是供创作参考的摘要层。

* **3.1 必须读取（写作前）**
  * 在撰写任意新章节正文前，必须读取：
    * `.novelkit/memory/character_state.md` — 主要角色当前状态、等级、位置、关系等。
    * `.novelkit/memory/foreshadowing.md` — 待收回伏笔、已收回伏笔、线索链与感情线。
  * 如目标章节已有章节提交文件（commit），在需要时也可参考：
    * `.novelkit/story/commits/chapter_{X}.commit.json` — 用于检查前一章事实记录。

* **3.2 必须更新（写作后 / 修订后）**
  * 每完成一个完整事件或关键章节后，必须在下一步写作前**增量更新** memory：
    * 补充或修订主要角色状态 → `.novelkit/memory/character_state.md`
    * 更新伏笔状态与线索链 → `.novelkit/memory/foreshadowing.md`
  * 更新规则遵循各自模板中的「增量更新铁律」，只能追加或修订，不得无故删掉旧信息。

* **3.3 章节提交（Chapter Commits）**
  * 每一章通过写作流程确认后，应生成对应的章节提交文件：
    * `.novelkit/story/commits/chapter_{X}.commit.json`
  * 章节提交记录这一章的**事实**（发生了什么、谁在场、状态如何变化、伏笔与线索情况等），是后续所有分析和可视化的唯一事实源。
  * 如需对既有事实进行重大更改，必须采用：
    * 「重写章节正文 → 生成新的 commit」的方式进行回溯修正，
    * 禁止静默篡改旧 commit 中的事实字段，只允许追加元信息（如健康标记、审查结果）。

# 附录：AI 写作避坑指南 (v2.0)

> 本附录定义了具体的词汇、句式禁忌和核心校准策略。凡是出现在这里的，一律杀无赦。

## 1. 语法与句式规范 (Syntax & Structure)

### 1.1 并列定语结构优化

* **规则**：禁止使用带有顿号的并列定语修饰宾语。
* **错误示例**：
  * 「那辆复古的、铺着红色绒布的人力黄包车」
  * 「一本厚重的、写满批注的笔记本」
* **修正建议**：调整语序或删除定语，例如：
  * 「那辆铺着红色绒布的复古人力黄包车」
  * 「一本写满批注的厚重笔记本」

### 1.2 标点符号优化

* **规则**：删除文本中多余的逗号，减少不必要的停顿，保持语句紧凑通顺。

---

## 2. 负向词库 (Negative Vocabulary / Blacklist)

> *以下词汇/句式带有强烈的「AI味」或翻译腔，严禁在正文中使用。*

### 2.1 严禁使用的词汇 (Strictly Prohibited Words)

* **神态与微动作**：
  * 瞬间、脸上带着笑意、显得有些兴奋、眼神热切、目光里毫不遮掩、XXX 的眼神。
  * 深吸一口气缓缓地、脸上堆满了笑、捏紧了拳头、指节因用力而发白。
  * 浮现出一抹、嘴角勾起一抹/一丝弧度/冷笑/玩味（请用具体动作代替，如「他笑了」）。
  * 死死盯着、瞳孔剧烈收缩、瞳孔收缩。
* **形容词**：
  * 锐利的、不容置疑、不易察觉、微不可察、纯粹、彻骨的冰寒。
  * 淬毒、冰冷、深邃、森然、轮廓分明的下颌、难以置信的、死寂。
  * 恐怖如斯。
* **高频虚词/连接词**：
  * 涟漪、轰然、死亡的阴影。

### 2.2 AI 味短语与句式 (AI-flavored Phrases)

* **滥用比喻**（每章比喻句不超过 3 个）：
  * 像一盆冰水、像一把精准的刻刀、像千年寒冰、像一根滚烫的钢针。
  * 像是根针、像一记重锤、像探照灯一样、像一把手术刀。
  * 像困住的 XXX、这 X 个字像 X 把锤子一下一下砸在 XXX 上。
  * XXX 的空气像是被 XXX 抽干了。
  * 宛如 / 仿佛 / 似乎 XXX。
* **陈词滥调**：
  * 倒吸一口凉气（除非真的冷）。
  * 剑眉星目 / 面如冠玉。
  * 深邃的眼眸 / 深邃的眸子 / 淬毒的眸子 / 冰冷的声音。
  * 空气仿佛凝固了、黑夜如墨 / 夜色如墨。
  * 心中翻滚 / 咆哮 / 绝望。
  * 死死地钉在 XX。
* **逻辑废话**：
  * 他相信 / 他知道。
  * 不得不说 / 毋庸置疑。
  * 电光火石之间 / 说时迟那时快。
  * 心中一动 / 心中一凛 / 心中暗道（尽量用行动展示心理）。

### 2.3 逻辑毒点 (Logical Pitfalls)

* **教科书式长难句**：超过 20 字的从句堆叠。  
  → **改为**：短句，口语化。
* **无意义的总结**：在对话或剧情后加一段「这意味着……」。  
  → **直接删除**。
* **过度解释**：角色说了一句话，旁白解释了三行。  
  → **相信读者的智商**。

## 4. 「去 AI 味」核心策略 (De-AI-ification Strategy)

### 4.1 注入「人性化」的不确定性

* **多用内心吐槽和现代梗**：打破 AI 「一本正经」风格，注入角色人格。
* **避免上帝视角解释**：严禁在剧情后添加「他知道……」「这意味着……」等多余旁白，通过行为和对话暗示。

### 4.2 强化独特的「作者风格」

* **建立角色口癖**：在正向白名单中明确主角或重要配角的口头禅，在润色时强制植入。
* **控制叙事节奏**：有意识地打破 AI 生成的均匀段落。多用短句和对话加速，只在需要渲染氛围时使用长句。
* **坚持核心法则**：在人工校准时，用项目的核心创作原则（如「体制内黑吃黑」「讽刺性杀人」）审视每一段剧情，确保骨架被填充了正确的血肉和灵魂。
